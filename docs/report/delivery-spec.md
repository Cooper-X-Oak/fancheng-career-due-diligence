# 交付规范

## 交付范围

本报告在 `docs/report/` 下形成一套可由 VitePress 构建和审阅的 Markdown 文档。每个报告文件聚焦一个独立调研问题，并在文件末尾列出检索痕迹、引用来源和信息缺口。

## 事实等级

| 等级 | 含义 | 使用方式 |
| --- | --- | --- |
| A | 官方网站、监管机构、公开数据库、企业官方招聘或官方公司页 | 用于确认主体、产品线、业务定位、全球网点、监管风险 |
| B | 招聘平台、LinkedIn、行业媒体、展会/协会、雇主评价平台 | 用于推断岗位、组织规模、雇主信号、行业位置 |
| C | X / Twitter、Reddit、B站、小红书、论坛与个人经验帖 | 用于识别口碑样本、压力场景、行业从业者叙事 |
| D | 二手市场报告搬运、SEO 聚合页、无作者页面 | 仅作为线索，不单独作为关键结论依据 |

## 引用与信息缺口

每章末尾包含：

- 本章检索问题。
- 本章检索平台。
- 主要证据。
- 信息缺口。
- 引用来源。

对于无法由公开资料确认的事项，统一表述为：“公开信息未能确认。由于该信息对决策重要，因此构成信息不透明风险。”

## 禁止内容

本报告不使用占位式、转交式或要求他人事后补录的模板内容。所有判断必须落在公开资料、合理推断或明确的信息不透明风险上。

## 报告边界

本报告不替代法律、劳动仲裁、税务或心理健康专业意见。报告只基于截至 2026-07-03 的公开信息和本机可访问平台检索结果，后续企业状态、招聘页面、社交平台内容和监管记录可能变化。

## 本文件检索痕迹

- 读取目标文件：`docs/goal-prompt.md`。
- 检索平台：本地文件、VitePress 配置、agent-reach doctor。
- 结果：确认目标要求每个文件独立成文、独立检索、给出来源和信息缺口。
